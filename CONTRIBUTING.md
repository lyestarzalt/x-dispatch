# Contributing to X-Dispatch

Thanks for your interest in contributing! This document outlines how to get started.

## Development Setup

1. Fork and clone the repository
2. Install dependencies: `npm install`
3. Start development: `npm start`

## Code Style

- Run `npm run check` before committing (typecheck + lint + format)
- Use conventional commits: `feat:`, `fix:`, `refactor:`, `chore:`, `docs:`
- Keep PRs focused on a single change

## Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add dark mode toggle
fix: resolve map layer displacement
refactor: simplify flight plan parser
docs: update installation instructions
chore: update dependencies
```

## Pull Request Process

1. Create a feature branch from `main`
2. Make your changes with clear commit messages
3. Run `npm run check` to verify code quality
4. Open a PR with a clear description of changes
5. Link any related issues

## Reporting Issues

- Use the issue templates for bugs and feature requests
- Include reproduction steps for bugs
- Check existing issues before creating duplicates

## Questions?

Open a discussion or issue if you need help getting started.
